The authoritative candidate workflow is:

```text
uv run python -m faultwitness_dev verify-fast
uv run python -m faultwitness_dev eval-changed
uv run python -m faultwitness_dev deploy-platform --candidate-sha <full-sha>
uv run python -m faultwitness_dev inspect-platform --candidate-sha <full-sha> --stability-seconds 900
uv run python -m faultwitness_dev eval-iteration I-0009 --candidate-sha <full-sha>
```

Raw kubeconfig, credentials, dumps, IdP/object-store exports, private URLs, and raw telemetry stay outside the public repository. Public evidence is limited to sanitized aggregates, exact restore digest equality, supply-chain summaries, and the complete candidate SHA.

- Unauthorized network, secret, schema, bucket-prefix, and dashboard access: 0.

All criteria apply conjunctively to one immutable candidate. Partial deployment, point-in-time readiness, unpaired denial tests, or an unbound private-server observation is not passing evidence. EVAL-G01-003 remains pending until the full pre-registered run completes.
