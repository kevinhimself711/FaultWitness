# I-0014 — Multi-Family ModelGateway and Compatible Channels

## Status

Planned; depends on I-0010 and I-0013.

## Objective

Provide one typed model boundary that proves real Qwen, DeepSeek, and GLM capabilities without overstating upstream independence.

## Scope

- Bailian live and NewAPI-compatible channels, exact model snapshots, capability discovery, route-policy versions, and trace linkage.
- Complete, structured-output, forced-tool, streaming, usage, timeout, retry, repair, and pre-output fallback behavior.
- Three repetitions of four live capabilities for each of three model families; offline fake-upstream conformance for NewAPI.

## Non-goals

No three-provider claim, live NewAPI evidence, diagnosis-quality comparison, architecture ablation, or cost optimization.

## Test and Eval

EVAL-G01-008 runs 36 Bailian live trials, deterministic stream reconstruction, schema validation, transient/permanent error matrices, malformed chunks, and post-first-chunk failure cases.

## Pass criteria

- Live capability trials: 36/36; structured/tool schema validity and stream reconstruction: 100%.
- Cross-model continuation after first visible chunk: 0.
- Every request records exact channel/model/route/usage/trace or a typed unavailable value.
- NewAPI wire suite passes while reports retain the no-live-redundancy limitation.
