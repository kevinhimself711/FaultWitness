# I-0004 — Architecture Trust Boundaries and Decisions

## Status

Completed on 2026-07-22.

## Objective

Freeze the system, container, data/control, deployment/trust, and state-ownership views so I-0005 can define contracts without reopening core boundaries.

## Scope

- Five mutually linked architecture documents and a machine-readable architecture source.
- Three mandatory engineering planes and five logical responsibility planes.
- Component responsibility, trust level, inputs, outputs, authoritative stores, and invocation boundaries.
- Six state-asset owners, typed Command/Event coordination, and data truth.
- Initial Threat Model with explicit zero-tolerance paths.
- Six accepted ADRs for planes, outbox, LangSmith, local artifacts, at-least-once delivery, and Action safety.
- Ten architecture failure/success walkthroughs required by the frozen Gate plan.

## Non-goals

- State transition tables, OpenAPI, AsyncAPI, and core type schemas; these belong to I-0005.
- Product, deployment, database, Agent, Sandbox, or action implementation.
- Selecting prompt, model, chunk, worker, lease, scaling, or training parameters.

## Dependencies

- I-0003 accepted Requirement and Evidence registry.
- Frozen G00 architecture and safety invariants.

## Test and Eval

- `make verify-fast`
- `make eval-changed`
- `uv run pytest tests/governance -q`
- EVAL-G00-004 architecture invariants and walkthrough completeness.

## Pass criteria

- All five architecture documents exist, contain Mermaid views, and cross-link.
- All three engineering and five logical planes are present.
- Every component declares responsibility, trust, inputs, outputs, and data truth references.
- State ownership is exact, unique, and matches declared write authority.
- Agent-to-action, Agent-to-Ground-Truth, cross-tenant, and unsandboxed-code bypasses are explicitly prohibited.
- W-ARCH-001 through W-ARCH-010 resolve only to known components.
- All six required ADRs are accepted and their indexed documents exist.
- No product implementation is added.

## Outcome

EVAL-G00-004 passed locally and in protected-main pull-request CI on Windows and Ubuntu. The machine-readable model contains 15 components, 9 stores, 6 unique state owners, 9 trust boundaries, 4 mandatory prohibited paths, and all 10 frozen walkthroughs.
