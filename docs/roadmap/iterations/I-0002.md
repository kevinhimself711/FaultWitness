# I-0002 — Governance Schemas, CLI, and Baseline CI

## Status

Completed on 2026-07-22.

## Objective

Turn the planning rules into executable repository governance before requirements and architecture assets expand.

## Scope

- JSON Schemas for PROJECT_STATE, Requirement, Claim, Iteration, Gate, Eval, and ADR assets.
- Machine-readable Gate and Iteration records plus empty Requirement and ADR registries.
- Canonical `uv run python -m faultwitness_dev <command>` CLI.
- Ruff, pytest, Markdown, UTF-8, local-link, schema, cross-reference, and changed-asset checks.
- Negative fixtures for missing fields, duplicate IDs, unknown references, missing behavior documentation, illegal status, and threshold reduction.
- SHA-pinned Windows and Ubuntu GitHub Actions baseline.
- Active GitHub ruleset for `main` after the baseline checks pass.
- Schema-validated Ruleset-as-code so the remote rule cannot silently drift from the reviewed policy.

## Non-goals

- Populate the JD/interview evidence catalog or requirements.
- Define the product architecture, state machines, or public interfaces.
- Add any API, agent, sandbox, data pipeline, or model implementation.

## Dependencies

- I-0001 repository and toolchain baseline.
- Frozen G00 Gate Master Plan.

## Test and Eval

- `make verify-fast`
- `make eval-changed`
- `uv run pytest tests/governance -q`
- Pull-request CI on Windows and Ubuntu.
- EVAL-G00-002 negative fixtures must fail for the intended reason.

## Pass criteria

- Every governed asset validates against its registered Schema.
- Cross-references reject nonexistent Gates, Iterations, and Requirements.
- Every frozen negative fixture is rejected without weakening a rule.
- Both canonical Make targets pass locally and in clean Windows/Ubuntu CI.
- `main` is covered by an active ruleset requiring pull requests and baseline checks.
- No product implementation is present.

## Outcome

All local and cross-platform checks passed. The reviewed Ruleset-as-code was deployed as active Ruleset `19545995` after the project owner approved public repository visibility in [AMD-0001](../../gates/G00/AMENDMENTS/AMD-0001.md). The Ruleset requires pull requests, both baseline platforms, current branches, stale-review dismissal, thread resolution, and blocks deletion and non-fast-forward updates.
