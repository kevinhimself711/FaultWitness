# I-0007 — Secure Bootstrap and Capability Baseline

## Status

In progress from the implementation-authorizing activation commit. This Iteration authorizes only the secure credential bootstrap and allowlisted read-only capability probes defined below; it does not authorize server mutation, deployment, model calls, or LangSmith calls.

## Objective

Replace the plaintext handoff path with project-scoped secret and host identities, then freeze a sanitized pre-deployment capability baseline.

## Scope

- Dedicated SSH Ed25519 identity, manually verified host-key pin, SOPS + Age policy, and stdin/API-only secret delivery.
- Verified same-value migration and explicit acceptance of the operator-declared long-lived server, Bailian, and LangSmith credentials before deleting and ignoring `envs.txt`.
- Sanitized inventory for Docker containers, listeners, networks, kernel/cgroup, KVM, GPU, storage, and required versions.

## Non-goals

No K3s installation, Docker modification, model request, LangSmith export, or product code.

## Test and Eval

EVAL-G01-001 executes repository/history/output secret canaries, existing-credential acceptance and server-login verification, host-pin negative cases, and deterministic capability-report generation. Bailian and LangSmith live validity remain owned by I-0013 and I-0014.

## Pass criteria

- Secret, host address, login identity, and reversible credential fingerprint findings in Git or public evidence: 0.
- Existing credential values are unchanged, accepted as long-lived, and protected outside Git; server login passes before `envs.txt` is deleted and precisely ignored.
- Dedicated SSH key and pinned host identity succeed; mismatch fails closed.
- Capability report is complete, sanitized, reproducible, and bound to the candidate SHA.
