# I-0001 — Repository Toolchain and Private Remote

## Status

Completed on 2026-07-22.

## Objective

Establish a reproducible repository and toolchain baseline before governance or product implementation begins.

## Scope

- Pin Python 3.12, Node.js 22.14.0, uv 0.10.12, and pnpm 11.9.0.
- Commit Python and Node workspace manifests and lockfiles.
- Provide `make verify-fast` and `make eval-changed` on Ubuntu and through a repository-local Windows shim.
- Create the private GitHub repository with `main` as the default branch.
- Record reproducibility and repository-boundary evidence in EVAL-G00-001.

## Non-goals

- Governance schemas and the canonical `faultwitness_dev` CLI.
- CI workflows and branch rulesets.
- Product packages, services, agents, sandboxes, or model code.
- Copying raw JD, interview, or upstream report material into this repository.

## Dependencies

- Frozen G00 Gate Master Plan.
- Git, Python 3.12, uv, Node.js 22, pnpm 11, and authenticated GitHub CLI.

## Implementation

1. Add version pins, manifests, and generated lockfiles.
2. Add a thin Makefile plus a Windows-local command shim.
3. Add temporary standard-library bootstrap checks; I-0002 replaces them with the canonical governance CLI.
4. Run both required commands and a clean-checkout smoke test.
5. Create and verify the private GitHub remote.

## Test and Eval

- `make verify-fast`
- `make eval-changed`
- Clean checkout smoke tests on the available Windows environment.
- GitHub metadata checks for visibility, remote URL, and default branch.
- A temporary, SHA-pinned GitHub Actions workflow executes the same checks on clean Windows and Ubuntu runners; the temporary validation branch is deleted after evidence capture.

## Pass criteria

- All pinned-version and lock files exist and parse through their native tools.
- Both required Make targets pass without a globally installed Make on Windows.
- No raw parent research material is tracked.
- Repository is private, `origin` targets it, and the default branch is `main`.
- No product implementation exists.

## Outcome

The repository/toolchain baseline passed on the local checkout, a fresh Windows clone, and clean GitHub-hosted Windows and Ubuntu runners. The remote is private, its default branch is `main`, and no product implementation was introduced.
