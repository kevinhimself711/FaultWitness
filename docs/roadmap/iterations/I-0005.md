# I-0005 — State Interface and Failure Contracts

## Status

In progress on 2026-07-22.

## Objective

Turn the I-0004 architecture baseline into machine-readable state, REST/SSE, Command/Event, type, error, and failure contracts without implementing product behavior.

## Scope

- Incident, Runtime Task, Agent Graph, and ActionTransaction transition tables.
- Deterministically generated and byte-validated Mermaid diagrams.
- OpenAPI 3.1 design for eight REST/SSE paths.
- AsyncAPI 3.0 design for eight internal Command/Event channels.
- Twenty-one core types with owner, store, version, field source, sensitivity, and invariants.
- Complete referenced Command, Event, error, and fixed failure catalogs.
- Conformance validation for ownership, reachability, terminal convergence, references, identity, idempotency, concurrency, and safety paths.

## Non-goals

- API server, client SDK, Pydantic model, message broker, reducer, database, or Action implementation.
- Concrete retry, lease, model, prompt, provider, or scaling values.
- Claiming runtime correctness from a design contract.

## Test and Eval

- `make verify-fast`
- `make eval-changed`
- `uv run pytest tests/contracts tests/governance -q`
- EVAL-G00-005 positive conformance and frozen negative safety mutations.

## Pass criteria

- Four machines have initial/terminal/owner/store and every transition has Actor, Guard, Preconditions, Command, Event, automatic flag, and failure semantics.
- Every required state is reachable and every non-terminal state can reach a terminal state.
- Terminal outgoing transitions, unknown references, and diagram drift are zero.
- R2 approval bypass, `UNCERTAIN` automatic retry, and stale-lease success commit paths are zero.
- OpenAPI and AsyncAPI parse and satisfy identity, idempotency, version conflict, resume, envelope, and error semantics.
- All 21 core types, 17 fixed failures, and referenced Command/Event/error IDs resolve.
- No product implementation is added.

## Outcome

Local EVAL-G00-005 checks pass. Four machines contain 52 states and 82 transitions; 8 REST/SSE paths, 8 AsyncAPI channels, 21 types, 34 Commands, 43 Events, 10 errors, 17 fixed failures, and all 10 walkthrough bindings are conformant. Protected-main Windows and Ubuntu evidence is pending.
