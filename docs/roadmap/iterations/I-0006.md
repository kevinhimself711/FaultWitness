# I-0006 — Gate Hardening and Candidate Audit

## Status

Completed on 2026-07-22 with a machine-enforced asset-only Gate closure protocol.

## Objective

Turn the G00 design baseline into a publicly reproducible, fail-closed candidate audit without implementing product behavior or closing the Gate.

## Scope

- Required Windows and Ubuntu verification plus a required Ubuntu audit status.
- Locked Python and Node dependency verification and CycloneDX 1.6 SBOM generation.
- Secret, absolute-local-path, license, source-ownership, and GitHub Action SHA checks.
- Browser rendering for every committed Mermaid block.
- PR and Issue templates, non-blocking scheduled external-link reporting, and evidence asset synchronization.
- Exactly fourteen frozen design walkthroughs mapped to the I-0005 contract bindings.
- Frozen `not_started` lifecycle support for the G01 handoff after G00 closes.
- A dedicated close-mode validator that rejects incomplete Iterations, non-passing Evals, open evidence, pending work, and waivers.
- An exact nine-asset closure boundary that rejects source, test, Schema, workflow, lockfile, and threshold changes.

## Non-goals

- Agent runtime, business API, model invocation, infrastructure deployment, sandbox, data pipeline, or runtime Eval.
- LangSmith trace evidence; it is explicitly not applicable because G00 has no runtime.
- Gate closure, architecture semantic changes, or threshold reductions.
