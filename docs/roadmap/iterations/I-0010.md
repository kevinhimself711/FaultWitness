# I-0010 — Executable Contracts and Transition Kernel

## Status

Planned; depends on I-0009 and does not activate persistence or APIs.

## Objective

Compile the G00 design authority into strict executable data and transition contracts without semantic drift.

## Scope

- Deterministic compiler and source digests for state machines, Commands, Events, types, errors, and failures.
- Twenty-one core types plus G01 Command, Outbox, Inbox, Checkpoint, Model, Trace, feedback, and stream support types.
- Four owner-specific transition services, explicit guard predicates, version 1.1.0, and compatibility checks.

## Non-goals

No database transport, API server, Agent graph, model invocation, or state-owner merger.

## Test and Eval

EVAL-G01-004 performs byte-stable regeneration, full transition conformance, mutation tests for owners/guards/versions/idempotency, and public-contract compatibility checks.

## Pass criteria

- G00 21 core types, 52 states, 82 transitions, 34 Commands, 43 Events, and 10 Errors: zero drift.
- Illegal transition, missing owner/guard/version/idempotency acceptance: 0.
- Unknown actor, predicate, schema, Command, or Event fails service startup.
- Existing REST/AsyncAPI breaking changes: 0.
