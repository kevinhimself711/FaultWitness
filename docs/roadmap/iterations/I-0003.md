# I-0003 — Requirements Evidence and Source Governance

## Status

Completed on 2026-07-22.

## Objective

Convert all in-scope JD and interview inputs plus the upstream report into a versioned, machine-verifiable requirement registry before architecture decisions begin.

## Scope

- Anonymous source catalog with content digests and explicit inclusion decisions.
- Fifty-seven testable requirements mapped to roles, architecture planes, Gates, and verification methods.
- Evidence matrix with exact requirement coverage and explicit evidence strength.
- Automated source-count, reference-integrity, Tier C, and matrix-coverage controls.
- Upstream, originality, and third-party boundaries.

## Non-goals

- Product architecture, state machines, interfaces, or runtime implementation.
- Reproduction of third-party research content in this repository.
- Claiming that a planned requirement has already been delivered.

## Dependencies

- I-0002 executable governance and protected-main baseline.
- Frozen G00 Gate Master Plan.
- The complete project-local source corpus outside this repository.

## Test and Eval

- `make verify-fast`
- `make eval-changed`
- `uv run pytest tests/governance -q`
- EVAL-G00-003 positive coverage and frozen negative-policy cases.

## Pass criteria

- Catalog counts are exactly 22 JDs, 84 included interview materials, 2 excluded empty interview materials, 1 technical reference, and 1 upstream report.
- All 57 requirements have a falsifiable verification method and mappings to roles, architecture planes, and Gates.
- Every requirement source reference resolves to an included source.
- No mandatory requirement relies only on Tier C evidence.
- The evidence matrix covers every requirement exactly once.
- No raw source text, source filename, local absolute path, credential, or personal trace is committed.

## Outcome

EVAL-G00-003 passed locally and in protected-main pull-request CI on Windows and Ubuntu. All 110 source records and 57 requirements validate, exact matrix coverage is enforced, and the publication scan is clean.
