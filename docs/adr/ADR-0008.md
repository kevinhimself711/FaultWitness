# ADR-0008: Route Three Model Families Through Bailian and Retain a NewAPI-Compatible Channel

Status: accepted

Date: 2026-07-22

## Context

The current Bailian credential can call Qwen, DeepSeek, and GLM model families. A separate NewAPI-compatible relay is planned, but no token is currently available. G01 needs enough real model evidence to validate structured output, tool calling, streaming, usage, retry, repair, fallback, and tracing while keeping public claims aligned with the actual upstream topology.

## Decision

Implement a channel-neutral ModelGateway with `BailianChannel` and `NewAPIChannel`. Bailian is the sole live upstream required by G01 and must run all three model families. NewAPI must pass the same protocol and failure suite against a controlled fake upstream; its live integration remains explicitly deferred until a token exists.

Every route records channel, model family, exact non-floating model identifier, capability flags, policy version, retry, fallback, usage, and trace correlation. Retries are limited to transient pre-output failures. Structured-output repair occurs at most once. Fallback may occur only before any streamed chunk reaches the caller; partial streams end as typed failures and are never stitched across models.

## Alternatives

- Call each model SDK directly from Agent nodes: rejected because capability, error, cost, and trace semantics would drift.
- Claim three providers based on three model families: rejected because all live calls share one upstream and failure domain.
- Block G01 until a NewAPI token exists: rejected because it adds no evidence for the platform foundation; the missing live channel remains a precise later start condition instead.

## Consequences

G01 proves model-family coverage and adapter compatibility, not upstream redundancy. Reports, Claims, demos, and interview narratives must retain that limitation. Later live NewAPI enablement requires credential bootstrap, exact-model discovery, the same capability suite, and an updated evidence manifest; it must not silently broaden a passing G01 claim.

## Migration and rollback

Channel configuration is versioned and defaults to Bailian-only routing. A failing or unavailable channel is disabled without changing request schemas. Route-policy changes require replayable trace evidence and regression tests for retry, repair, fallback, and partial-stream behavior.
