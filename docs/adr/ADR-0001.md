# ADR-0001: Integrate Three Engineering Planes Behind Five Logical Responsibility Planes

Status: accepted  
Date: 2026-07-22

## Context

The project must demonstrate Agent application depth, multi-tenant runtime/infra depth, and data/eval/training depth. Treating the latter two as optional appendices would produce a polished demo rather than the integrated system required by the evidence registry. Treating every concern as one deployable service would instead destroy ownership, security, and independent evaluation boundaries.

## Decision

Plan and accept the project as three mandatory engineering planes:

1. Agent Application.
2. Agent Runtime/Infra.
3. Data/Eval/Training.

Within runtime behavior, use five logical responsibility planes: Experience, Control, Data, Execution, and Evaluation. Components may participate in more than one logical plane but belong to exactly one engineering delivery plane. The three engineering planes are independently testable and deployable, yet none is optional for final project completion.

## Alternatives

- Agent-only MVP with Infra and Training as future work: rejected because it contradicts the integrated outcome and leaves core interview depth unimplemented.
- One monolithic Agent service: rejected because it creates overlapping state ownership and privileged execution paths.
- Separate repositories and roadmaps: rejected for G00 because cross-plane contracts and Gate evidence would drift.

## Consequences

Every Requirement and Gate maps to logical planes and target roles. Interfaces and stores remain decoupled, while the final Claim registry cannot call the project complete without all three engineering planes. P2 indicates admission order and evidence prerequisites, not an optional project component.

## Migration and rollback

Changing a component's engineering plane or removing a mandatory plane requires a superseding ADR, updated Requirement/Evidence mappings, contract compatibility analysis, and rerun of architecture and affected Gate Evals. Rollback restores the last accepted mapping and deployment ownership; no data migration exists at G00.
