# ADR-0003: Use LangSmith as a Strong Trace and Evaluation Dependency

Status: accepted  
Date: 2026-07-22

## Context

The project needs attributable model/tool trajectories, dataset and experiment lineage, badcase analysis, and repeatable Eval evidence. Building all of this as a custom platform during the same project would dilute Agent and Infra depth, while treating observability as optional would make interview claims unauditable.

## Decision

Use LangSmith as the authoritative Agent Trace/Eval service for trace, dataset, experiment, and evaluator lineage. All exports pass through typed sanitization and correlation instrumentation. A bounded local buffer supports transient outages.

If LangSmith is unavailable, explicitly qualified read-only diagnosis may continue with local correlation evidence. New write actions fail closed, evaluation runs are incomplete, and no Gate may close until authoritative evidence is present.

## Alternatives

- OpenTelemetry-only custom Eval store: rejected as the G00 default because it requires recreating dataset/experiment/evaluator capabilities.
- Best-effort tracing: rejected because missing traces would invalidate safety and quality Claims.
- Send raw model transcripts: rejected because secrets, PII, and private reasoning must not be retained.

## Consequences

LangSmith availability becomes an explicit operational dependency and cost. The system must implement redaction, export allowlists, outage buffering, and reconciliation. OpenTelemetry remains the operational telemetry standard; LangSmith is not a replacement for service metrics/logs/traces.

## Migration and rollback

Instrumentation emits a provider-neutral internal envelope so a future backend can be dual-written and replayed. A replacement requires lineage parity, evaluator reproducibility, privacy review, and rerun of affected Eval Claims. Rollback resumes the previous exporter and replays the bounded buffer.
