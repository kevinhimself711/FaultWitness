# AMD-0001 — Public Repository Visibility

## Status

Accepted on 2026-07-22 by explicit project-owner instruction.

## Original constraint

The frozen G00 plan required the GitHub repository to remain private while the repository and baseline governance were established.

## Amended constraint

The source repository is public. Runtime environments, credentials, raw JD/interview materials, restricted source documents, private traces, and local evidence remain private and must never be committed.

This amendment replaces only the repository-visibility assertions in the frozen G00 plan. It does not modify any architecture, security, evidence, state-machine, interface, CI, Eval, or quality threshold.

## Rationale

The project owner explicitly selected public visibility so GitHub Free can enforce the reviewed branch Ruleset. The final blueprint already permits publication after source, license, and secret review; this amendment advances that visibility transition while preserving the full G00 audit in I-0006.

## Preconditions and safeguards

- Full Git history was scanned before publication for credential patterns, private keys, local absolute paths, and raw research-material names.
- No credential, private key, local source path, or raw JD/interview artifact was found.
- The only research-name match was the governance checker's denylist itself.
- `main-governance` is active on the default branch and requires pull requests plus Windows and Ubuntu baseline checks.
- I-0006 must still run the frozen source, license, SBOM, secret, and attribution audits; this amendment is not a waiver for them.
