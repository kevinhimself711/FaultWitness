# 原创性与项目贡献边界

## 继承的研究输入

岗位 JD、面经和上游报告只用于确定市场要求、常见追问和深度基准。它们不是 FaultWitness 的产品实现、架构证明或性能结果。

## FaultWitness 的原创贡献

- 将通用 Agent 能力落入可复现的微服务事故闭环，同时保留 Agent 应用、AI Infra 和算法训练的一体化演进路径。
- 以五个架构平面和 G00-G10 Gate 组织实现顺序、失败语义、评估资产与提交边界。
- 将来源、需求、Claim、Iteration 和 Eval 建成机器校验的证据链，而不是只写叙述性方案。
- 后续将实现独立状态所有权、类型化命令与事件、受控副作用、租户隔离、数据谱系和可归因训练实验。
- 所有质量、可靠性、性能和成本结论都必须来自 FaultWitness 自有的版本化实验，不能从调研材料推断。

## 面试陈述约束

可以陈述“需求来自哪些证据类别”和“项目如何响应这些要求”，不能把第三方经历、招聘描述或上游总结表述为本人生产经验。尚未通过对应 Gate 的能力只能陈述为规划，不能列入 verified Claim。
