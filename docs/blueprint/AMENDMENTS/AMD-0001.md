# AMD-0001 — Clarify G01 Model Topology and Live-Evidence Boundary

Status: accepted

Date: 2026-07-22

Owner: project owner

## Context

The frozen G00 blueprint described GLM, DeepSeek, and Qwen/Bailian adapters and later summarized the G01 exit as three providers. The available credential actually reaches three model families through one Bailian upstream. A NewAPI-compatible relay is planned, but its token is not yet available. Treating model-family diversity as provider independence would overstate the evidence.

## Amendment

- G01 uses one live `BailianChannel` for Qwen, DeepSeek, and GLM model families.
- G01 also implements and tests a `NewAPIChannel` against an OpenAI-compatible fake upstream.
- The Gate proves multi-family routing and two-channel contract compatibility, not cross-provider availability or failure independence.
- NewAPI live evidence is deferred until a token exists and becomes a start condition for the first later Runtime Gate that claims live channel redundancy.
- Exact model snapshots, routes, retries, fallbacks, and usage must remain traceable.

## Impact

This amendment corrects claim precision and does not change the four state owners, trust boundaries, public paths, Gate ordering, or security thresholds. No migration or replay work is required because no runtime implementation or stored model route exists yet.
